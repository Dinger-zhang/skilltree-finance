# 决策日志与待解决问题

最后更新：2026-06-07

## 1. 已确认的重要决策

### 决策 1：不从小学生数学开始验证

原因：

1. 当前不易接触小学生样本。
2. K12 监管和信任门槛高。
3. 小学数学周期长，不适合早期快速验证。
4. 先用成人/大学生场景验证学习机制更稳妥。

### 决策 2：第一个 MVP 场景选“零基础财务报表学习”

原因：

1. 对计算机同学足够陌生。
2. 知识结构清晰。
3. 容易做技能树和推理链。
4. 学习结果容易测量。
5. 2 周左右可完成实验。

### 决策 3：使用 Codex 开发本地 MVP

技术方向：

```text
Python + Streamlit + SQLite + YAML + pandas
```

原因：

1. 开发快。
2. 适合实验原型。
3. 本地运行即可。
4. 便于 Codex 读写和修改。

### 决策 4：早期不做完整平台

暂不做：

```text
登录
支付
云部署
移动端
多学科
复杂 UI
排行榜
社交功能
```

原因：当前最重要的是验证学习机制。

### 决策 5：项目不能停留在测评系统

最新关键纠偏：

当前系统如果只是前测、做题、后测、报告，就更像测评系统，不是真正学习系统。

应升级为：

```text
原子知识点
推理链
引导式学习
学生自我解释
掌握验证
路径推荐
```

### 决策 6：v0.2 优先重构知识图谱和推理式学习页面

当前优先级：

```text
1. 微知识点重构
2. 推理链字段设计
3. 推理式学习页面
4. expected_reasoning_points 判断
5. 学生解释记录
```

暂缓：

```text
管理员页面
UI 美化
复杂报告
大模型接入
```

### 决策 7：商业路径先成人/大学生，再职业培训/机构，再考虑 K12

理由：更易验证、更少合规负担、更容易找到样本和付费场景。

## 2. 产品哲学记录

### 核心产品哲学

本项目不以灌输知识点为目标，而以构造可推理的知识路径为目标。

每个知识节点必须明确回答：

```text
它依赖哪些前置节点？
它可以推出哪些后续节点？
它解决什么认知问题？
它容易产生什么误解？
学生如何证明自己真正掌握？
```

### 与普通 AI 学习助手的区别

普通 AI：

```text
你不会，我给你讲。
```

SkillTree：

```text
你不会，我判断你缺哪个前置概念；
我把它拆到更小；
让你用已知知识推出新知识；
再验证你是否真的能迁移。
```

## 3. 已被调整或否定的方向

### 否定 1：直接宣传“AI 替代老师”

原因：容易引起抵触，也不准确。

更好的表述：

> AI 将高质量一对一教学中的诊断、反馈、路径规划和复习机制规模化。

### 否定 2：一开始做全学科平台

原因：范围过大，无法验证单点价值。

替代：先做财务报表单模块。

### 否定 3：一开始做复杂大模型智能体

原因：会混淆效果来源，不知道效果来自大模型讲解还是技能树路径。

替代：先用规则和结构化节点验证学习机制，再接入 LLM。

### 否定 4：先做 UI 美化和商业 Demo

原因：容易变成路演项目，没有真实学习效果。

替代：先做实验数据，再做展示版。

## 4. 当前待解决问题

### 问题 1：如何判断一个知识点已经足够原子？

初步标准：

```text
是否只解决一个认知问题？
是否能用 1 个核心问题表达？
是否有明确前置节点？
是否能推出明确后续节点？
是否能设计一道掌握验证题？
```

仍需继续完善。

### 问题 2：expected_reasoning_points 如何评分？

第一版：关键词/要点匹配。

后续：引入 LLM 判断学生解释是否覆盖关键推理点。

需要考虑：

```text
如何避免学生关键词命中但不理解？
如何处理同义表达？
如何提供可解释反馈？
```

### 问题 3：推理链是否一定线性？

当前第一版可线性展示，便于 MVP。

后续应支持图结构：一个节点可推出多个节点，一个节点也可能依赖多个前置节点。

### 问题 4：财报知识图谱由谁审核？

当前由项目负责人和 AI 共同整理。

后续如果商业化，需要财经专业人士或内容专家审核。

### 问题 5：如何证明“推理式学习”优于普通资料学习？

需要对照实验：

```text
实验组：SkillTree 推理式学习
对照组：普通文章/视频/ChatGPT 自学
```

比较：

```text
前后测提升
单位时间提升
延迟测试保持率
迁移题正确率
主观路径清晰度
```

### 问题 6：第一个付费产品怎么包装？

候选：

> 7 天 AI 技能树财务报表入门营。

仍需确定：

```text
价格
目标用户
学习承诺
交付形式
是否需要社群陪伴
```

### 问题 7：是否扩展到 AI 工具学习和论文精读？

初步判断：可以作为第 2、3 个垂直领域。

但前提是财报场景验证出技能树学习机制有效。
